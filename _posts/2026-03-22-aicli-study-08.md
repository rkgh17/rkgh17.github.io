# 10장 - Gemini CLI로 프로젝트 운영하기

GEMINI.md는 Claude Code의 CLAUDE.md, Codex의 AGENTS.md와 마찬가지로 프로젝트와 관련된 핵심 정보를 정리해 AI작업을 돕기 위한 문서이다. 다만, Gemini CLI 에서는 이 파일이 행동을 강제하는 규칙 파일이 아니라, 작업 맥락을 이해시키기 위한 참고용 컨텍스트 문서이다.


## 10.1 - GEMINI.md 파일 관리

### GEMINI.md의 위치와 활용 범위


1. 프로젝트 루트 GEMINI.md - 프로젝트 컨텍스트 문서
2. 서브폴더 GEMINI.md - 선택적 참고 문서

### GEMINI.md 작성 방식과 기본 구조

- 기준
	- AI가 실행해야 할 규칙을 나열하지 않는다.
	- 프로젝트를 처음 접하는 사람이 읽어도 이해할 수 있도록 작성한다.

- 실무에서는 보통 다음과 같은 항목을 중심으로 작성한다.
	1. 프로젝트 개요
	2. 주요 워크플로
	3. 협업 및 작업 지침
	4. 주의 사항

### GEMINI.md 작성 팁

1. 기본 문서부터 작성
2. 문서 다듬기 요청 활용
3. 사용자 검토 및 직업 수정
4. 실전 대화에서 효과 확인하기

### GEMINI.md 반영 상태 점검과 컨텍스트 관리

1. 작업 요청에서 컨텍스트를 분명히 전달
2. GEMINI.md 반영 여부는 작업 결과로 확인
3. 문서 변경 시 자동 재적용을 기대하지 않기
4. 컨텍스트 문서를 주기적으로 점검


## 10.2 - 세션 관리


### 세션 저장하고 복원
- 세션 상태 저장 : `/chat save <이름>`
- 저장된 세션 목록 확인 : `/chat list`
- 세션 복원 : `/chat resume <이름>`
- 세션 삭제 : `/chat delete <이름>`
- 대화 내용 내보내기 : `/chat share <파일명>`
- 긴급 복구 : `/restore`
