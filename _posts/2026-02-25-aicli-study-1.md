# 1일차

## 터미널 기반 에이전틱 AI

**에이전틱 AI (Agentic AI)** 는 사용자의 지시를 입력받은 뒤, 스스로 하위 작업을 분해하고 명령을 실행하며, 필요하면 결과를 검증 및 보완하는 자율형 AI 작업자(Autonomous AI Agent)를 뜻하며 다음과 같은 특성을 갖는다.

- **맥락 유지** : 프로그램의 요구 사항과 중간 산출물을 기억하고 지속적으로 활용
- **장기 실행** : 여러 단계로 구성된 복잡한 업무를 순차적으로 처리
- **환경 제어** : 파일을 읽고 쓰고 실행하여 실제 코드베이스를 조작

위같은 특성 덕분에 개발자는 반복작업을 AI에게 위임하고, 설계, 의사결정, 품질 검증 같은 고차원적 업무에 집중할 수 있다. 이처럼 AI와 사람이 자연스럽게 역할을 나누며 개발 흐름을 만들어가는 방식을 **바이브 코딩(Vibe coding)** 이라고 한다.  
터미널 기반 에이전틱 AI는 터미널 환경에서 실행되며, 로컬 파일 시스템과 개발 환경에 직접 접근할 수 있어 실제 프로젝트 구조를 그대로 다룰 수 있으며, 대표적은 AI 터미널 도구로는 Claude Code, Codex CLI, Cemini CLI 가 있다.

## 사용자 유형 별 추천 플랜 및 비용 전략

(2025-12월 기준)

| 사용자 유형    | 추천 도구/플랜                                 | 이유                              |
| --------- | ---------------------------------------- | ------------------------------- |
| 입문자/취미개발자 | Gemini CLI 무료 티어                         | 가장 관대한 무료 사용량 제공, 초기 비용 부담 없음   |
| 전문 개발자    | Claude Code Pro 또는 Max                   | 장시간 코딩, 대규모 분석, 정교한 코드 작업에 안정적  |
| 팀/스타트업    | Claude Code Team 또는 OpenAI Team/Business | 팀 관리 기능, 권한 설정, 협업 기능 제공        |
| 엔터프라이즈    | 각 도구의 Enterprise 플랜                      | 보안, 감사 로그, 컴플라이언스 등 기업 환경 요구 반영 |

AI 터미널 도구 사용량은 누적되기 때문에 비용을 관리하는 전략을 초기에 세우면 효율성을 크게 높일 수 있다. 다음 네가지 전략이 추천된다.

1. 무료 티어 최대 활용
	- Gemini CLI를 먼저 사용해 자신의 작업 패턴과 실제 필요 용량을 파악.
2. 하이브리드 전략 운영
	- 가벼운 작업은 Gemini CLI로 진행하고, 초기 프로토타이핑, 문서 생성, 정밀 코드 생성은 Claude Code 또는 Codex CLI로 진행하는 등 역할을 분리한다.
3. 사용량 모니터링 및 최적화
	- 불필요한 LLM 호출 줄이기, 로컬 캐시를 활용해 프롬프트 재사용하기 등으로 비용을 안정적으로 관리한다.
4. 팀 플랜 적극 활용
	- 팀 단위 개발에서는 개인별 구독보다 팀 플랜 구독이 더 경제적이며, 권한 관리, 프로젝트 공유, 정책 기준 통일 등의 장점도 함께 제공한다.
